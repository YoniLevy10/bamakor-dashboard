# סקריפט הודעות — וואטסאפ לדיירים ו־SMS לצוות

מסמך זה מתאר **את כל הטקסטים** שהמערכת שולחת ללקוחות קצה (דיירים בוואטסאפ, מנהלים ועובדים ב־SMS).  
ניתן לערוך כאן ניסוחים ואז לעדכן את המפתחים בקוד (או לבקש מהמפתח להעתיק משפה לשם).

**מקרא:** טקסטים בסוגריים מרובעים `[כך]` הם משתנים מהמערכת (שם בניין, מספר תקלה וכו').

---

## חלק א׳ — הודעות וואטסאפ לדיירים

מקור עיקרי: `app/api/webhook/whatsapp/route.ts`  
גם: `app/api/close-ticket/route.ts` (כשסוגרים תקלה)

---

### FLOW: שליחת מיקום (GPS)

**TRIGGER:** דייר שולח הודעת מיקום בוואטסאפ (`messageType` = location).

**MESSAGE:**

```
קיבלנו מיקום! לדיווח תקלה כתבו תיאור הבעיה בטקסט 📍
```

---

### FLOW: שליחת אנשי קשר (כרטיס)

**TRIGGER:** דייר שולח אנשי קשר (`messageType` = contacts).

**MESSAGE:**

```
קיבלנו את אנשי הקשר. לדיווח תקלה כתבו את כתובת הבניין או סרקו את קוד ה-QR 📇
```

---

### FLOW: מדבקה (סטיקר)

**TRIGGER:** דייר שולח מדבקה.

**MESSAGE (אם אין סשן פעיל — לא זוהה בניין בשיחה):**

```
שלחו את שם הרחוב לדיווח תקלה 😊
```

**MESSAGE (אם יש סשן פעיל — כבר נבחר בניין וממתינים לתיאור):**

```
קיבלנו 🙂 כתבו כשתוכלו את תיאור התקלה או פרטים נוספים.
```

---

### FLOW: הודעת קול

**TRIGGER:** דייר שולח הודעת קול / אודיו.

**MESSAGE:**

```
🎙️ קיבלנו הודעת קול, אך לא נוכל לעבד אותה.

בשביל לדווח תקלה:
1️⃣ כתבו את התקלה בטקסט
2️⃣ או שלחו תמונה של התקלה

רוצים להתחיל? סרקו את קוד ה-QR בבניין או כתבו את כתובת הבניין.
```

---

### FLOW: תמונה — צורפה לתקלה קיימת (הצלחה)

**TRIGGER:** יש תקלה פעילה המקושרת לסשן, והתמונה צורפה בהצלחה.

**MESSAGE:**

```
✅ התמונה התקבלה בהצלחה וצורפה לתקלה. צוות הטכנאים יטפל בבקשתך בהקדם.
```

---

### FLOW: תמונה — נכשלה הצרפה אך התקלה נשמרת

**TRIGGER:** ניסיון צירוף תמונה לתקלה פעילה נכשל (הורדה / אחסון / DB).

**MESSAGE:**

```
⚠️ לא הצלחנו להוסיף את התמונה, אך התקלה שלך תקבלה.

נעדכן כשיהיה טיפול.
```

(שימו לב: במקור בקוד מופיע «תקבלה» — אם רוצים תיקון לשון: «התקבלה».)

---

### FLOW: תמונה לפני טקסט (בניין כבר נבחר, אין עדיין תקלה)

**TRIGGER:** יש סשן עם `project_id`, אין עדיין תקלה בטקסט, והתמונה נשמרת לצירוף בהמשך.

**MESSAGE:**

```
🖼️ קיבלנו את התמונה!

כדי לצרף אותה לתקלה, כתבו עכשיו בקצרה את תיאור התקלה בטקסט.
```

---

### FLOW: תמונה — צורפה לתקלה אחרונה (חלון זמן קצר)

**TRIGGER:** אין סשן פעיל, אבל נמצאה תקלה אחרונה של אותו מספר (לא סגורה) בטווח זמן קצר — והצירוף הצליח.

**MESSAGE:** (זהה להצלחת «תמונה לתקלה קיימת»)

```
✅ התמונה התקבלה בהצלחה וצורפה לתקלה. צוות הטכנאים יטפל בבקשתך בהקדם.
```

---

### FLOW: תמונה — צירוף לתקלה אחרונה נכשל

**TRIGGER:** ניסיון צירוף לתקלה אחרונה נכשל.

**MESSAGE:**

```
⚠️ לא הצלחנו להוסיף את התמונה, אך התקלה שלך התקבלה.

אם תרצו, נסו לשלוח את התמונה שוב או פנו למנהלת הבניין.
```

---

### FLOW: תמונה בלי הקשר (אין סשן / אין תקלה אחרונה רלוונטית)

**TRIGGER:** התקבלה תמונה בלי הקשר לסשן או לתקלה.

**MESSAGE:**

```
🖼️ קיבלנו את התמונה!

כדי לצרף אותה לתקלה, צריך קודם:
1️⃣ סרקו QR בבניין
2️⃣ כתבו תיאור התקלה
3️⃣ אז שלחו תמונה

השתדלו!
```

---

### FLOW: סרטון או מסמך

**TRIGGER:** דייר שולח סרטון או קובץ (לא תמונה).

**MESSAGE (סרטון):**

```
קיבלנו סרטון 📎

בשביל דיווח תקלה, אנא שלחו תמונה או כתבו תיאור.

רוצים להתחיל? סרקו את קוד ה-QR בבניין.
```

**MESSAGE (קובץ):** אותו ניסוח עם המילה «קובץ» במקום «סרטון».

---

### FLOW: תגובה (Reaction) ללא טקסט

**TRIGGER:** סוג הודעה reaction וללא גוף טקסט.

**MESSAGE:**

```
קיבלנו את התגובה 🙂 לדיווח תקלה כתבו את הפרטים כאן.
```

---

### FLOW: סריקת QR — קוד START לא תקין

**TRIGGER:** ההודעה מתחילה ב־`START_` אבל הפורמט לא תקין.

**MESSAGE:**

```
פורמט קוד ה-QR לא תקין. אנא סרקו שוב את הקוד או פנו למנהלת הבניין.
```

---

### FLOW: סריקת QR — לא נמצא פרויקט לקוד

**TRIGGER:** קוד `START_BMK...` תקין אבל אין פרויקט במערכת עם אותו קוד.

**MESSAGE:**

```
❌ לא הצלחנו לזהות את הפרויקט.

נסו שוב:
1️⃣ סרקו את QR מחדש
2️⃣ או כתבו את כתובת הבניין (רחוב ומספר)
3️⃣ או צרו קשר למנהלת הבניין
```

---

### FLOW: סריקת QR — הצלחה (פתיחת סשן + בקשת תיאור)

**TRIGGER:** קוד `START_BMK...` תקין, נמצא פרויקט, נפתח סשן.

**MESSAGE:**

```
ברוכים הבאים! פתחנו תקלה חדשה עבורכם ב[שם הפרויקט][אופציונלי:  (בניין [מספר בניין מהקוד])]. כתבו בקצרה את הבעיה 📝
```

דוגמה ללא בניין משני בקוד:  
`ברוכים הבאים! פתחנו תקלה חדשה עבורכם בהרצל 12. כתבו בקצרה את הבעיה 📝`

---

### FLOW: שאילת סטטוס תקלה

**TRIGGER:** ההודעה מזוהה כשאלה על סטטוס / עדכון (למשל «מה קורה עם התקלה», «עדכון?»).

**MESSAGE (יש פנייה פתוחה):**

```
תקלה #[מספר תקלה] בסטטוס: [חדשה / משויכת / בטיפול / ממתינה לחלקים]
נוצרה ב־[תאריך ושעה בעברית מקומית]. אנחנו על זה! 🔧
```

**MESSAGE (אין פנייה פתוחה):**

```
כרגע אין פנייה פתוחה ממספר זה.
כדי לפתוח תקלה חדשה כתבו את שם הרחוב או סרקו את קוד ה-QR 🔧
```

---

### FLOW: ברכה / שיחת חולין (בלי סשן)

**TRIGGER:** מילות ברכה קצרות («שלום», «תודה», «בוקר טוב» וכו') ואין סשן פעיל (לפני שחזור זיכרון בניין).

**MESSAGE:**

```
שלום! כדי לדווח תקלה, כתבו את שם הרחוב של הבניין או סרקו את קוד ה-QR 😊
```

---

### FLOW: ברכה עם סשן פעיל (כבר נבחר בניין)

**TRIGGER:** אותן מילות ברכה, אבל כבר יש סשן פתוח.

**MESSAGE:**

```
שלום! כשתוכלו, כתבו בקצרה את תיאור התקלה כאן בצ׳אט 📝
```

---

### FLOW: טקסט באנגלית (ללא סשן)

**TRIGGER:** ההודעה מזוהית כאנגלית בעיקרה.

**MESSAGE:**

```
Hello! To report an issue, please write your building address or scan the QR code 😊
```

---

### FLOW: מספר טלפון או שם («שמי יוסי», מספר סלולרי)

**TRIGGER:** שורה שנראית כמספר טלפון ישראלי או «שמי …» / «אני …».

**MESSAGE:**

```
תודה! כדי לדווח תקלה, כתבו את שם הרחוב של הבניין 😊
```

---

### FLOW: אימוג׳י / אישור קצר בלי סשן

**TRIGGER:** אימוג׳י בלבד או מילות אישור קצרות מאוד (👍, כן, וכו') ואין סשן.

**MESSAGE:**

```
שלחו את שם הרחוב לדיווח תקלה 😊
```

---

### FLOW: דחיפות בלי הקשר לכתובת

**TRIGGER:** מילות דחיפות («דחוף», «אש», וכו') אבל הטקסט **לא** מסווג כחיפוש כתובת.

**MESSAGE:**

```
קיבלנו! זו נראית תקלה דחופה.
כתבו את שם הבניין ונטפל מיד 🚨
```

---

### FLOW: אימוג׳י / אישור — עם סשן פעיל

**TRIGGER:** אימוג׳י קצר או אישור, ויש סשן.

**MESSAGE:**

```
קיבלנו 🙂 נמשיך לטפל — כתבו עוד פרטים אם צריך.
```

---

### FLOW: פרט דירה / קומה (עם סשן)

**TRIGGER:** שורה שמזוהית כפרט דירה בלבד («דירה 5», «קומה 3» וכו') ויש סשן עם בניין.

**MESSAGE:**

```
קיבלנו את פרט הדירה/קומה ✅ נצרף אותו לפנייה כשתשלחו את תיאור התקלה.
```

---

### FLOW: זיכרון בניין — חזרה אוטומטית אחרי פעילות קודמת

**TRIGGER:** אין סשן פעיל, אין בחירה ממתינה, אבל יש היסטוריה — המערכת משחזרת את הבניין האחרון לטלפון.

**MESSAGE:**

```
ברוכים הבאים! פתחנו תקלה עבורכם ב[שם הפרויקט]. כתבו בקצרה את הבעיה 📝
```

---

### FLOW: בחירת בניין מהרשימה (אחרי חיפוש — המשתמש שלח 1 / 2 / 3)

**TRIGGER:** היה רשימת התאמות, המשתמש שלח מספר תקין.

**MESSAGE:** (זהה לזרם QR מוצלח מבחינת ניסוח)

```
ברוכים הבאים! פתחנו תקלה חדשה עבורכם ב[שם הפרויקט שנבחר]. כתבו בקצרה את הבעיה 📝
```

---

### FLOW: מספר לא תקין כשיש רשימת בחירה

**TRIGGER:** יש `pending selection`, המשתמש שלח מספר שלא בטווח.

**MESSAGE:**

```
אנא השיבו רק עם מספר האפשרות המתאים: 1, 2 או 3.
```

---

### FLOW: תזכורת — צריך לבחור מספר

**TRIGGER:** יש רשימת בחירה ממתינה, וההודעה לא מספר ולא «כתובת חדשה» לפי הכללים.

**MESSAGE:**

```
⚠️ השיבו רק עם מספר האפשרות: 1, 2 או 3
```

---

### FLOW: חיפוש כתובת — טקסט לא נראה כמו כתובת

**TRIGGER:** אין סשן, אין בחירה ממתינה, והטקסט לא עובר את מסנן «נראה כמו כתובת».

**MESSAGE:**

```
לא הצלחנו לזהות את הבניין.

📍 כדי שנוכל לאתר אותו, כתבו את כתובת הבניין (רחוב ומספר)

או:
1. סרקו את קוד ה-QR בבניין
2. פנו למנהלת הבניין לקבלת קוד הגישה
```

---

### FLOW: חיפוש כתובת — לא נמצאו תוצאות

**TRIGGER:** הטקסט נחשב כתובת־לייק, אבל אין התאמות בפרויקטים.

**MESSAGE:** (זהה לניסוח למעלה — «לא הצלחנו לזהות…»)

```
לא הצלחנו לזהות את הבניין.

📍 כדי שנוכל לאתר אותו, כתבו את כתובת הבניין (רחוב ומספר)

או:
1. סרקו את קוד ה-QR בבניין
2. פנו למנהלת הבניין לקבלת קוד הגישה
```

---

### FLOW: חיפוש כתובת — התאמה יחידה

**TRIGGER:** נמצאה התאמה אחת לחיפוש.

**MESSAGE:**

```
ברוכים הבאים! פתחנו תקלה חדשה עבורכם ב[שם הפרויקט]. כתבו בקצרה את הבעיה 📝
```

---

### FLOW: חיפוש כתובת — כמה התאמות (רשימה ממוספרת)

**TRIGGER:** נמצאו 2–3 התאמות.

**MESSAGE (שלד קבוע + שורות דינמיות):**

```
מצאנו כמה בניינים תואמים:

1. [שם פרויקט 1] ([כתובת אם קיימת])
2. [שם פרויקט 2] ...
3. ...

📌 להמשך, השיבו רק עם מספר האפשרות: 1, 2 או 3
```

---

### FLOW: שגיאה טכנית בשמירת רשימת הבחירה

**TRIGGER:** יצירת `pending_selection` נכשלה.

**MESSAGE:**

```
תקלה טכנית. אנא סרקו את קוד ה-QR בבניין או פנו למנהלת הבניין.
```

---

### FLOW: פרט דירה בלי סשן / בלי בניין

**TRIGGER:** שורת דירה בלבד, בלי סשן פעיל.

**MESSAGE:**

```
כדי לדווח תקלה כתבו את שם הרחוב של הבניין
```

---

### FLOW: עדכון לתקלה קיימת (כפילות תוך 5 דקות)

**TRIGGER:** נפתחה לאחרונה תקלה פתוחה מאותו מספר ואותו פרויקט — ההודעה נוספת לתיאור במקום תקלה חדשה.

**MESSAGE:**

```
קיבלנו את העדכון והוספנו לפנייה #[מספר תקלה] ✅
```

---

### FLOW: נפתחה תקלה חדשה מהצ׳אט (אחרי תיאור)

**TRIGGER:** נוצרה תקלה חדשה בוואטסאפ; נשלחת אישור לדייר.

**MESSAGE:**

```
התקלה התקבלה בהצלחה.[אופציונלי: שורה חדשה «בניין: [מספר]»]
מספר הפנייה שלך: [מספר רץ]

💡 אפשר גם לשלוח תמונה של התקלה — זה יעזור לנו לטפל בה מהר יותר.

נעדכן כשיהיה טיפול.
לפתיחת תקלה נוספת — כתבו בקצרה את הבעיה (אין צורך לסרוק QR שוב).
```

---

### FLOW: סגירת תקלה — עדכון לדייר

**TRIGGER:** מנהל/מערכת סוגרים תקלה; לדייר יש `reporter_phone` בוואטסאפ.

**מקור:** `app/api/close-ticket/route.ts`

**MESSAGE:**

```
✅ שלום! התקלה שדיווחת בבניין [שם הבניין או «הבניין» אם חסר] טופלה וסגורה.

אם יש בעיה נוספת, ניתן לפנות אלינו בכל עת 🙏
```

## חלק ב׳ — הודעות שאינן «לדייר בשגרה» (למודעות)

### בדיקת חיבור וואטסאפ מהגדרות

**נשלח אל:** מספר המנהל שהוגדר בהגדרות (לא בהכרח דייר).  
**מקור:** `app/api/settings/test-whatsapp/route.ts`

**MESSAGE:**

```
✅ בדיקת חיבור מהמערכת — [שם הלקוח במערכת או «המערכת»]
```

### הערה: הודעת וואטסאפ לעובד (לא פעילה)

ב־`app/api/assign-ticket/route.ts` קיימת **הערה בלבד** (קוד מושבת) להודעת וואטסאפ לעובד בעת שיוך תקלה. כרגע **לא נשלחת** — נשלח SMS בלבד.  
הטקסט שהיה מתוכנן (ארכיון):

```
הוקצתה לך תקלה חדשה.

פרויקט: [שם]
פנייה: [מספר]
תיאור: [תיאור או «ללא תיאור»]
```

## חלק ג׳ — הודעות SMS למנהל ולעובדים

הערה: גוף ה־SMS נבנה בקוד; שם השולח ב־019SMS יכול להיות לפי `sms_sender_name` מהלקוח או ברירת מחדל.

### SMS למנהל — נפתחה תקלה חדשה מוואטסאפ

**TRIGGER:** נוצרה תקלה מוואטסאפ; יש יעד למנהל (מספר מנהל בלקוח / בפרויקט / משתנה סביבה).

**מקור:** `app/api/webhook/whatsapp/route.ts`

**MESSAGE:**

```
נפתחה תקלה חדשה
פרויקט: [שם פרויקט]
[אופציונלי: בניין: [מספר]]
תקלה: #[מספר תקלה]
תיאור: [תיאור התקלה כולל שורת דירה אם נשמרה]
מדווח: [מספר וואטסאפ של הדיווח]
כניסה למערכת:
[כתובת האתר מהגדרת NEXT_PUBLIC_APP_URL]/tickets
[שם הלקוח/חברה]
```

---

### SMS לעובד משוייך — נפתחה תקלה חדשה מוואטסאפ

**TRIGGER:** לפרויקט יש `assigned_worker_id` עם טלפון.

**מקור:** `app/api/webhook/whatsapp/route.ts`

**MESSAGE:**

```
תקלה חדשה ב[שם פרויקט]
#[מספר תקלה]
[תיאור]
מדווח: [מספר]
[כתובת האתר]/tickets
[שם לקוח]
```

---

### SMS לעובד — הוקצתה לו תקלה (שיוך ידני / מערכת)

**TRIGGER:** קריאת API לשיוך תקלה לעובד.

**מקור:** `app/api/assign-ticket/route.ts`

**MESSAGE:**

```
תקלה חדשה הוקצתה לך
פרויקט: [שם פרויקט]
תקלה: #[מספר]
תיאור: [תיאור או «ללא פירוט»]
כניסה למערכת:
[כתובת האתר]/tickets
[שם לקוח]
```

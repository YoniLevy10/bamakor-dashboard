# ארכיון — לוגיקת WhatsApp הישנה (הוסרה ב־2026)

הקובץ `route.ts` כולל כיום **שתי זרימות בלבד**:

1. **סריקת QR** — `START_*` → ברכה + פרויקט → תיאור תקלה → יצירת טיקט → (אופציונלי) תמונה.
2. **טקסט חופשי בלי סשן** — אם הטקסט לא נראה כמו כתובת/חיפוש בניין → הודעת "לא הצלחנו לזהות את הבניין"; אחרת חיפוש בניין, כולל בחירה 1/2/3 כשיש כמה תוצאות → תיאור → טיקט → תמונה.

## מה הוסר מקוד הייצור (לשחזור: היסטוריית Git לפני הקומיט הרלוונטי)

- שאילתות סטטוס פנייה (`מה הסטטוס`, וכו') והשבות ייעודיות
- ברכות/אנגלית/אימוג׳י/דחיפות/שורת טלפון כשלבי ניתוב נפרדים
- שחזור סשן אוטומטי מ"בניין אחרון" בלי סריקה מחדש (`restoreSessionFromLastProjectPhone`)
- שמירת דירה בנפרד לפני תיאור (`parseApartmentDetailOnly` + עדכון `sessions.pending_apartment_detail`)

התנהגויות אלה נשמרות רק כאן כתיעוד; לאט לאט אפשר להחזיר מ־`git` לפי צורך.

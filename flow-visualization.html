⚠️ Error Handling Scenarios

Image Download Fails (Step 1)
Fallback: Ticket preserved ✅ → Send "Image failed but ticket OK" → Return 200
Storage Upload Fails (Step 2)
Fallback: Ticket preserved ✅ → Send "Image failed but ticket OK" → Return 200
DB Insert Fails (Step 3)
Fallback: Ticket preserved ✅ → Send "Image failed but ticket OK" → Return 200
Building Search Returns 0 Results
Guidance: "Try different address / Use QR / Contact manager" → Return 200 (user can retry)
Pending Selection Expires
Auto-clear after 10 min → Send "Try again" → Return 200 (user starts fresh)
Invalid Numeric Selection
Reminder: "Please send 1, 2, or 3" → Return 200 (pending still active)
Message Send Fails (Non-Blocking)
Log error, continue flow → Return 200 anyway (best effort)
Critical DB Error
Log error → Return 500 (signals external system to retry)
